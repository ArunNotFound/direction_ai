# Writing Skills

Format: YAML frontmatter (name, description, tier, triggers) + body.
Compatible with agentskills.io hosts (Hermes, Claude Code, others).

Rules of craft (superpowers lineage + Load Line discipline):
- **Description is the router.** Write it as "Use when …, before …" —
  trigger conditions first, capability second. Bad descriptions cause both
  missed loads and noisy loads.
- **Index, not encyclopedia.** A skill states laws and gates; long reference
  material lives in linked files the agent reads on demand. Durable rules go
  in PROTECTED sections; probabilistic tips stay editable.
- **One skill, one job.** If a skill needs an "and also" heading, split it.
- **Falsifiable where possible:** state the command or check that proves the
  skill was followed.
- **Iron-law voice for tier-0/1:** "violating the letter is violating the
  spirit" — no soft language on gates.
- New skills enter through skill-evolution-gate like any edit.
