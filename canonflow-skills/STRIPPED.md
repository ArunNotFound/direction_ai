# What Was Stripped, and Why

**From gstack (70+ skills → 4 concepts kept):** all iOS skills, browser/Chrome
automation, Supabase, deploy/ship/land pipelines, gbrain sync, design-review
family, office-hours, CEO/eng review theatrics — product-startup ops, not
library stewardship. Hook scripts tied to Claude Code paths replaced with
host-portable prose rules. `careful`+`guard` merged into one `guardrails`.

**From superpowers (14 skills → 6 concepts kept):** git-worktrees (Hermes
sandboxes supersede), finishing-a-development-branch (folded into
commit-provenance), brainstorming (steward does this with a frontier model in
chat, not via skill), dispatching-parallel-agents + subagent-driven-development
(folded into agent-governance — the governor owns dispatch), receiving/
requesting-code-review (folded into verification + governance).

**From SkillOpt (runtime → 0 code kept, 2 disciplines kept):** the entire
Python training loop, checkpoints, benchmarks, WebUI — we adopt the *contract*
(held-out validation gate, bounded edits, protected sections), not the
machinery. Known SkillOpt risks — validation overfitting, optimizer-target
leakage — are carried into banyan-bench and skill-evolution-gate as explicit
cautions rather than inherited silently.
